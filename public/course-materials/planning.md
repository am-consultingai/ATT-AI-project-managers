# Planning Skill Template

This planning template captures a general product and implementation planning workflow that can be reused across web projects.

## Purpose
Use this document to define:
- the product vision and executive summary
- the business logic and goals
- a phased R&D plan with testable milestones
- UX/UI requirements
- deployment targets and publish strategy

## Planning Workflow
1. Executive Summary
   - Describe the product in one or two paragraphs.
   - Highlight the user value, platform, and launch target.
   - Keep it concise and clear for non-technical stakeholders.

2. Business Logic
   - Define what problem the product solves.
   - List the core features and user outcomes.
   - Include any success criteria or business goals.

3. R&D Plan
   - Break the work into phases.
   - Assign each phase a status flag: `not implemented`, `in progress`, `implemented`.
   - Make each phase testable with clear outcomes.

   Example phases:
   - Phase 1: Project Setup
   - Phase 2: Core Functionality
   - Phase 3: Feedback / Edge Cases
   - Phase 4: UI and Responsiveness
   - Phase 5: Deployment Preparation

4. UX/UI Requirements
   - Define the visual and interaction goals.
   - Include accessibility, responsiveness, and user feedback requirements.
   - Keep requirements simple and focused on key screens.

5. Deployment and Publishing
   - Specify the hosting target and deployment path.
   - Include simple, step-by-step instructions for non-technical users.
   - Document any required hosting services or platforms.

## Status Flag Usage
- `not implemented` — phase defined but not started.
- `in progress` — work has begun and is actively being developed.
- `implemented` — phase is complete and verified against tests.

## Reuse Guidance
- Copy this template into new project documentation files.
- Tailor the phases to the project type (web app, service, landing page).
- Keep statuses updated as work progresses.
- Use plain language for business and non-technical stakeholders.
